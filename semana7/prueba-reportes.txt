Al realizar una petición HTTP utilizando el método GET a la dirección http://localhost:3000/reportes, el servidor responde con un código de estado 200 OK y devuelve un arreglo vacío representado por dos corchetes [].

Esto sucede porque la aplicación utiliza un arreglo en memoria llamado 'reportes' para almacenar la información de forma temporal. Al iniciar el servidor por primera vez, este arreglo no contiene ningún elemento. Una vez que se envíen datos a través de peticiones POST, este espacio se irá llenando con los objetos correspondientes a cada reporte ciudadano.
